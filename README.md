\# A/B Test Analysis (SQL, PostgreSQL)



\## Описание проекта



Проведён анализ результатов A/B теста с целью оценки влияния изменений продукта на ключевые бизнес-метрики.



Фокус анализа:

\- конверсия пользователей

\- монетизация (ARPU, средний чек)

\- итоговая выручка



\---



\## Задачи



\- Проверить корректность распределения пользователей (A/B split)

\- Рассчитать ключевые продуктовые метрики

\- Оценить влияние изменений на поведение пользователей

\- Проверить статистическую значимость различий

\- Сформулировать бизнес-рекомендацию



\---



\## Стек



\- SQL (PostgreSQL)

\- CTE

\- агрегатные функции

\- оконные функции

\- Z-test (проверка статистической значимости)

\## Анализ результатов



\### 1. Проверка сплита

Распределение пользователей между группами близко к 50/50 → тест корректен.



\---



\### 2. Конверсия

Группа B показывает незначительный рост конверсии:

+0.12 п.п.



Однако разница минимальна.



\---



\### 3. Монетизация



\#### ARPU

\- A: 99.14  

\- B: 79.34  



снижение \~20%



\---



\#### Средний чек

\- A: 88.82  

\- B: 94.36  



рост среднего чека в группе B



\---



\### 4. Выручка

\- A: 48 377  

\- B: 40 621  



снижение \~16%



\---



\## Статистическая проверка



Проведён Z-test для проверки значимости разницы в конверсии:



\- z-score ≈ -0.066



Разница статистически незначима (|z| < 1.96)



\---



\## Интерпретация



Несмотря на небольшой рост конверсии:

\- эффект не является статистически значимым

\- ARPU и выручка существенно снижаются



изменение ухудшает бизнес-метрики



\---



\## Вывод



Изменение не рекомендуется к внедрению.



Причины:

\- отсутствие статистически значимого роста конверсии

\- снижение ключевых метрик монетизации

\- падение общей выручки



\---



\## Бизнес-гипотеза



Рост конверсии может быть связан с привлечением менее платёжеспособных пользователей, что приводит к снижению ARPU.
